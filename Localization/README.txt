Разработал Лобович Артем

Локализатор поддерживает выдачу результатов типа string, AudioClip и может быть расширен до других типов.
Для расширения, необходимо создать компонент и унаследоваться от LocalizationProvider<T>, где вместо Т указать нужный тип. 
Также необходимо переопределить свойство Localization и указать массив компонентов, которые хотите использовать, унаследованный от LocalizedContainer<T>.


Как пользоваться

	1. Добавьте на сцену компонент Language
	2. В папке Resources создайте через контекстное меню нужный файл локализации (LocalizedAudioClip или LocalizedText или свой)
	3. В файле укажите значение ключа и нужные переводы в массиве. В массиве указывайте переводы в порядке, согласно договоренности (например, 0 - русский, 1 - английский и т.д.)
	4. На нужный компонент AudioSource или Text повесьте компонент Localizator. На другие компоненты (например Image) Localizator повесить не удастся без необходимого расширения.
	5. В поле Key вводите значение ключа. При совпадении значения ключа с ключом в файлах локализации, в соответствующее поле будет установлено значение ключа в зависимости от выбранного языка (работает только в редакторе).
	6. Можно использовать несколько файлов локализации для организации рабочего процесса (например, переводы отдельных слов в отдельном файле, переводы отдельных фраз в другом файле и т.д.). Названия файлов не важны.
	7. Для использования из скрипта, обратитесь к компоненту Localizator и вызовите метод ChangeKey если хотите изменить значение ключа в реалтайме.
	8. При переключении языка в компоненте Language все элементы с компонентом Localizator изменяют свои значения на актуальные автоматически (в т.ч. в режиме редактора).
	9. Для переключения языка вызовите в компоненте Language метод SetLanguage с порядковым номером языка в качестве аргументов. Метод не выполнится, если порядковый номер будет больше, чем установлено языков в enum LanguageCode.
